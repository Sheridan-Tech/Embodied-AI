# Embodied AI Notes Repository

All rights reserved unless otherwise noted.

This repository is for collecting, organizing, and consolidating notes related to embodied AI, digital twinning, faster-than-real-time simulation, robotics, and related physical-AI systems.

The goal is to turn scattered research notes, project references, technical observations, implementation ideas, source links, and experiment logs into a single structured repository that can support future development work.

## Purpose

This repository serves as a central knowledge base for projects involving:

* Embodied AI systems
* Robotics
* Digital twins
* Faster-than-real-time simulation
* Reinforcement learning for physical systems
* Sim-to-real transfer
* Robot perception and control
* Physical AI architectures
* Sensor-driven autonomy
* LLM-assisted robotics workflows
* Memory, self-improvement, and adaptive behavior in robots

The repository is intended to preserve useful information while making it easier to compare approaches, identify reusable patterns, and turn research into practical implementation plans.

## Repository Goals

The main goals of this repository are to:

1. Consolidate fragmented notes into one organized location.
2. Capture source material, links, videos, papers, tools, and implementation references.
3. Document project ideas and architectural concepts for embodied AI systems.
4. Track digital twin and simulation workflows.
5. Explore faster-than-real-time simulation as a way to accelerate training, testing, and validation.
6. Compare robotics frameworks, simulators, hardware platforms, sensors, and AI models.
7. Preserve design decisions, open questions, risks, and lessons learned.
8. Convert raw notes into cleaner project briefs, build plans, and technical documentation.

## Documentation Approach

Notes should be written so they are useful later, not just understandable in the moment.

Whenever possible, each note should include:

* Summary
* Source links
* Key claims or findings
* Implementation relevance
* Hardware or software dependencies
* Risks and limitations
* Open questions
* Next steps

For project-specific notes, include:

* Project name
* Objective
* Current status
* Hardware requirements
* Software requirements
* Simulation requirements
* Control architecture
* Training approach
* Testing plan
* Known unknowns
